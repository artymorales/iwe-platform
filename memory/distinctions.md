# Distinctions — Ключевые различения

> Различения — это концептуальные границы, которые важны для принятия решений
> в IWE. Если различение не используется регулярно — оно не нужно.
> Добавляются только по мере возникновения в практике.

## Текущие различения

### Base vs DS vs Pack
- **Base** — принципы и форматы (что неизменно в рамках платформы).
- **DS** — производное: планы, код, решения (меняется каждую неделю).
- **Pack** — паспорт предметной области (появляется, когда знаний по теме
  накапливается достаточно для формализации).

### Открытие vs Работа vs Закрытие
- **Открытие** — планирование, согласование, загрузка контекста.
- **Работа** — исполнение.
- **Закрытие** — фиксация результата, capture, commit.

### Capture vs Draft
- **Capture** — готовое знание, можно сразу сохранить.
- **Draft** — сырая мысль, требует доработки.

### Flash (лёгкая модель) vs Pro (тяжёлая модель)
- **Flash** — быстрые задачи: рефакторинг, чтение, форматирование, рутина.
- **Pro** — стратегические сессии, архитектурные решения, сложный анализ.

### Sync по commit vs Sync по контексту
- **Sync по commit** — между вкладками/сессиями. Агент в новой вкладке не видит изменений из этой вкладки, пока они не закоммичены.
- **Sync по контексту** — внутри одной сессии. Агент помнит всё что обсуждали в этом чате.
- **Правило:** перед `startTask` или открытием новой вкладки — commit в affected репозиториях.
