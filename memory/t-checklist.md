# T-чеклист (single source для Close)

> Линейный, не требующий суждения. Каждый пункт: owner, trigger, symptom-if-skipped.
> Идемпотентно, календарно, автопилот.

## Session Close (Quick Close ≤15 мин, или по триггеру «закрывай»)

| # | Действие | Owner | Trigger | Symptom-if-skipped |
|---|----------|-------|---------|--------------------|
| T1 | Commit всех изменений (`git add -A && git commit`) | агент | сессия с изменениями | незакоммиченные файлы, потеря работы |
| T2 | Push (по триггеру «заливай»/«запуши»/«закрывай») | агент | явный триггер | recovery не гарантирован |
| T3 | Capture-to-Pack: появилось ли знание для сохранения? | агент | любая сессия | знание теряется |
| T4 | Отчёт: что сделано, что дальше | агент | любая сессия | следующая сессия стартует с потерей контекста |

**Исключения:** сессии ≤15 мин БЕЗ изменений файлов → T1-T4 пропускаются (только устное summary).
