# Протокол Работы (Work)

> Применяется во время выполнения задачи.
> Цель: сохранять фокус и фиксировать знания по мере появления.

## Во время работы

### Capture-to-Pack (постоянно)
На каждом логическом рубеже проверять: есть ли знание для записи?
- Правило (1-3 строки) → `memory/distinctions.md`
- Доменное наблюдение → `~/ds-knowledge-index/captures/`
- Сырая мысль → `~/ds-knowledge-index/drafts/`

Формат: `## YYYY-MM-DD: Тема` + что узнал + откуда.

### Self-correction
Заметил расхождение с планом или прошлыми решениями?
- Остановиться.
- Сказать пользователю: что не так, где, как исправить.
- Не исправлять молча.

### Коммуникация
- В начале ответа — краткий план (1-2 предложения).
- Если задача идёт не по плану — сказать сразу.
- Закончил шаг — скажи, что сделано и что дальше.
