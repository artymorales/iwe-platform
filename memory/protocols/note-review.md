# Протокол Note-Review — разбор заметок

> **Триггер:** Команда «разбери заметки» / «note-review» / конец дня (перед Day Close).
> **Роль агента:** R1 Стратег.
> **Бюджет:** ~10-15 мин.
> **Адаптация:** Pi agent + Aethon (нет Telegram-бота, нет Экстрактора, нет Dissatisfactions.md).
> **Источник:** FMT `roles/strategist/prompts/note-review.md` — 11 шагов.

---

## Концепция

За день накапливаются сырые заметки (мысли, идеи, баги, «надо бы»).
Note-Review разбирает их: классифицирует, маршрутизирует в нужные документы, архивирует.

**Принципы:**
1. Note-Review **НЕ принимает решений** — классифицирует и предлагает
2. Предложения — рекомендации, требуют подтверждения пользователя
3. Знание идёт в captures (не в Pack — Экстрактора нет)
4. Обработанные заметки архивируются (не удаляются)

---

## Вход

- `~/ds-strategy/inbox/fleeting-notes.md` — сырые заметки
- Контекст: WeekPlan, WP-REGISTRY, последний DayPlan, captures

**Формат fleeting-notes.md:**
```markdown
# Fleeting Notes

## YYYY-MM-DD

**Заголовок заметки**
<sub>HH:MM</sub>
Текст заметки — что пришло в голову

---

**Другая заметка**
<sub>HH:MM</sub>
Кратко: суть мысли
```

- `**жирный заголовок**` = новая, не разобрана
- `Обычный заголовок` = обработана
- `~~Зачёркнутый~~` = шум (тест, дубль, случайное)

---

## 7 категорий

| # | Категория | Критерий | Куда |
|---|-----------|----------|------|
| 1 | **НЭП** | Дискомфорт, «хочу изменить», эмоция о системе | → предложение в конец WeekPlan (секция «НЭП») |
| 2 | **Задача** | Конкретное действие, «сделать X», баг | → предложение в DayPlan (завтра) или WeekPlan |
| 3 | **Знание** | Паттерн, различение, правило — применимо вне реализации | → `~/ds-knowledge-index/captures/` (кандидат) |
| 4 | **Черновик** | Зерно для поста/статьи, рефлексия с концептами | → рекомендация в WeekPlan (секция «Черновики») |
| 5 | **Личные данные** | Контакт, телефон, email, токен | → `~/ds-strategy/personal/` (если создан) |
| 6 | **Идея** | Размытое «было бы круто», без конкретики | → пометить 🔄, вернуться при стратегировании |
| 7 | **Шум** | Тест, дубль, случайное, уже сделано | → зачеркнуть `~~текст~~` |

**Правила классификации:**
- НЭП + Задача → приоритет НЭП (эмоция важнее действия)
- Ссылки без контекста → Шум
- Срочная задача (срок «сегодня», «завтра») → Задача с пометкой срока
- Личные/бытовые задачи → Задача (не отбрасывать как «нерелевантные»)

## Процесс (11 шагов)

### Шаг 1. Прочитать заметки

Прочитать `~/ds-strategy/inbox/fleeting-notes.md`.
Выделить все **жирные** заметки — это новые.

Если новых нет → сказать «Новых заметок нет» и завершить.

### Шаг 2. Проверить актуальность

Перед классификацией — не устарела ли заметка?

Сверить с коммитами за сегодня:
```bash
git -C ~/ds-strategy log --since="00:00" --all --oneline
```

- Задача из заметки уже сделана (есть коммит или РП done)? → Шум
- Проблема уже решена? → Шум
- Покрыто существующим РП (grep WP-REGISTRY)? → Шум, указать «Реализовано: РП #N»

### Шаг 3. Классифицировать

Для каждой новой заметки — определить категорию (1–7).

### Шаг 4. Пометить заметки

Обновить `fleeting-notes.md`:

| Категория | Было | Стало |
|-----------|------|-------|
| НЭП, Задача, Знание, Черновик, Личные данные | `**Заголовок**` | `Заголовок` (снять жирный) |
| Идея | `**Заголовок**` | `**Заголовок** 🔄` (оставить жирный + маркер) |
| Шум | `**Заголовок**` | `~~Заголовок~~` |

### Шаг 5. Записать в целевые документы

**НЭП → WeekPlan** (секция «НЭП» в конце):
```markdown
- [ ] **НЭП:** «краткий текст заметки» (из Note-Review YYYY-MM-DD)
```

**Задача → DayPlan (завтра) или WeekPlan:**
```markdown
- [ ] «краткий текст заметки» (из Note-Review)
```

**Срочная задача (сегодня/завтра) → DayPlan.** Остальное → WeekPlan.

### Шаг 6. Записать знание

Для категории «Знание» → `~/ds-knowledge-index/captures/`:
```markdown
## YYYY-MM-DD: Краткое описание

Что узнал: [из заметки]
Откуда: fleeting-notes.md, YYYY-MM-DD HH:MM
Контекст: [из какой ситуации]
```

Если знание — различение (1-3 строки, применимо везде) → `~/iwe-platform/memory/distinctions.md` вместо captures.

### Шаг 7. Рекомендация в черновик

Для категории «Черновик» → рекомендация в WeekPlan:
```markdown
- [ ] **Черновик:** «краткий текст» → рекомендую название: [предлагаемое]. Концепты: [список]
```

**Note-Review НЕ создаёт черновики сам** — только рекомендует. Пользователь решает при просмотре WeekPlan.

### Шаг 8. Проверить inbox

Беглый осмотр `~/ds-strategy/inbox/`:
- Нет ли аномалий? (неизвестные файлы, мусор)
- Если да → лог в отчёт (шаг 9)
- **НЕ чистить** — очистка inbox при Week Close

### Шаг 9. Показать отчёт

```
**Note-Review (DD мес):** N заметок обработано
- Задачи: [список] → DayPlan / WeekPlan
- Знание: [список] → captures / distinctions
- НЭП: [список] → WeekPlan
- Черновики: [список] → рекомендации
- Идеи 🔄: [список] → ждут стратегической сессии
- Шум: [список] (тест, дубль, уже сделано)
```

Спросить: *«Применить предложения? (да / выборочно / нет)»*

### Шаг 10. Архивировать и очистить

**10a. Добавить в архив:**
```bash
cat >> ~/ds-strategy/archive/notes/Notes-Archive.md << 'EOF'

## YYYY-MM-DD — Note-Review

Заголовок заметки
<sub>дата, время</sub>
**Категория:** НЭП / Задача / Знание / ...
Текст заметки

---
EOF
```

**10b. Очистить fleeting-notes.md:**
- Удалить все обработанные заметки (не-жирные, зачёркнутые)
- Оставить: шапку, заметки с 🔄, новые жирные (если появились во время обработки)

**10c. Проверить:** прочитать fleeting-notes.md → убедиться, что очищено.

### Шаг 11. Закоммитить

```bash
git -C ~/ds-strategy add -A
git -C ~/ds-strategy commit -m "note-review YYYY-MM-DD: N заметок"
git -C ~/ds-strategy push

# Если были записи в captures:
git -C ~/ds-knowledge-index add -A
git -C ~/ds-knowledge-index commit -m "note-review YYYY-MM-DD: captures"
git -C ~/ds-knowledge-index push

# Если были записи в distinctions:
git -C ~/iwe-platform add -A
git -C ~/iwe-platform commit -m "note-review YYYY-MM-DD: distinctions"
git -C ~/iwe-platform push
```
