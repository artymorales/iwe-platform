# Study Pipeline — конвейер входящего учебного материала

> **Триггер:** слот «Мышление письмом» в DayPlan, добавление нового материала в очередь.
> **Роль агента:** фасилитатор — выбрать, задать вопросы на пересказ, направить в capture.
> **Бюджет:** ~1–2h на материал.
> **Источник:** WP-31 (Н13: недостаточный входящий поток). Сверка с FMT — аналога нет,
>   учебный поток описан в FMT через note-review + creative-pipeline, но без
>   отдельного конвейера. Мы добавляем входной рукав.

---

## Концепция

Материал для изучения (статьи, видео, курсы) попадает в очередь → изучается с
активным пересказом → знание извлекается через `capture.sh` → попадает в творческий
конвейер (`creative-pipeline.sh`).

**Отличие от творческого конвейера:** Study Pipeline — **входной рукав**.
Творческий конвейер начинается с capture и ведёт к публикации.
Study Pipeline начинается с сырого материала и ведёт к capture.

```
Внешний материал → [Study Pipeline] → capture → [Creative Pipeline] → publish
                   reading-list          capture.sh   creative-pipeline.sh
                   пересказ
                   archive
```

---

## Процесс (5 шагов)

### Шаг 1. Select — выбор материала

Агент показывает очередь из `~/ds-knowledge-index/inbox/reading-list.md`:

```
Доступные материалы:
  🟡 Learning Path FMT — Material 2 [IWE] ~2h (TTL: W28)
  🟡 Как я учил LLM держать задачу [AI-agents] ~0.5h (TTL: W27)

Что берём в слот?
```

Правила выбора:
- 🔴 (TTL истекает) > 🟡 > 🟢
- Соответствие бюджету слота (~1–2h)
- Релевантность текущим проектам

### Шаг 2. Study — изучение с пересказом

Чтение/просмотр с остановками. Агент задаёт вопросы:
- *«Что понял? Перескажи своими словами.»*
- *«Какие ключевые идеи? Выдели 3–5.»*
- *«Где это применимо в твоих проектах?»*
- *«Что противоречит твоему текущему пониманию?»*

**Принцип:** не пассивное чтение, а активная пересборка.

### Шаг 3. Capture — захват знания

```bash
bash ~/iwe-platform/scripts/skills/capture.sh "Тема" "Источник" "Контекст"
```

Если материал тянет на пост:
```bash
bash ~/iwe-platform/scripts/skills/capture.sh --type draft "Заголовок" "Источник"
```

### Шаг 4. Archive — архивирование

1. Добавить строку в таблицу «Архив» в reading-list.md
2. Удалить строку из таблицы «Очередь»
3. Обновить статус: `⏳ queue` → запись в архиве с датой и ссылкой на capture

### Шаг 5. Pipeline check — проверка конвейера

```bash
bash ~/iwe-platform/scripts/skills/creative-pipeline.sh check
```

Если captures накопились (>10) — `resolve-captures`.

## Интеграция с другими протоколами

| Протокол | Как связан |
|----------|-----------|
| **Day Open** | Сканирует очередь, показывает доступные материалы. Предупреждает о TTL |
| **Capture (capture.sh)** | Принимает знание из Study Pipeline. Вход в творческий конвейер |
| **Creative Pipeline** | Получает capture → ведёт к draft → Pack → publish |
| **Note-Review** | Параллельный входной рукав (fleeting-notes → capture) |
| **Week Close** | Проверка: сколько материалов изучено, сколько captures создано |

## Добавление материала в очередь

Агент или пользователь добавляет строку в `reading-list.md`:

```markdown
| ⏳ queue | 🟡 | Название | Домен | URL | ~Время | YYYY-MM-DD | W{N} |
```

**Правила:**
- Guard: ≤10 в очереди. При превышении — приоритизировать или отсеять.
- TTL опционален. Ставится, если материал привязан к событию/неделе.
- Домен — для группировки: IWE, AI-agents, Архитектура, Карьера, Английский.
