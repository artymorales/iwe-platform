# Протокол стратегической сессии

> Применяется в `strategy_day` (по умолчанию — **понедельник**).
> Если это **первая сессия вообще** (нет Strategy.md) — см. раздел «Initial flow».
> Если сессия регулярная — см. «Weekly flow».

---

## Initial flow (день-0, нет артефактов)

> Цель: запустить систему с нуля. Без ревью прошлого — его нет.

### 1. Цели (5 мин)

Спросить пользователя:
- «Кем хочешь быть через год?»
- «Чему хочешь научиться?»
- «Какие 2-3 крупные цели на ближайшие 3-6 месяцев?»

Записать в `~/ds-strategy/docs/Strategy.md`:
- Видение (1 год)
- Цели на горизонт (3-6 месяцев)
- Принципы (что важно)

### 2. Неудовлетворённости (5 мин)

Спросить:
- «Что сейчас мешает? Где разрыв между текущим и желаемым?»
- «Что регулярно раздражает или забирает энергию?»

Записать в `~/ds-strategy/docs/Dissatisfactions.md`.

### 3. Первый WeekPlan (10 мин)

На основе целей + НЭП предложить 3-5 РП на первую неделю. Для каждого:
- Название (существительное-артефакт)
- Бюджет (часы)
- Артефакт-критерий

Записать в `~/ds-strategy/current/weekplan-W{N}.md`.

### 4. Сохранить и закоммитить

## Weekly flow (есть Strategy.md и/или прошлый WeekPlan)

> Длительность: ~1.5–2ч. Модель: **pro** (тяжёлая).

### Шаг 1. Контекст (перед началом)

Прочитать:
- `~/iwe-platform/memory/day-rhythm-config.yaml` (strategy_day, настройки)
- `~/ds-strategy/docs/Dissatisfactions.md` (активные НЭП)
- `~/ds-strategy/docs/Strategy.md` (фокус месяца, результаты, R)
- `~/ds-strategy/docs/WP-REGISTRY.md` (все РП, статусы)
- `~/ds-strategy/current/weekplan-W{N-1}.md` (прошлая неделя, если есть)
- `~/ds-strategy/inbox/` (входящие)

### Шаг 2. Ревью НЭП (5–10 мин)

- Пройти список НЭП: какие сняты? какие добавились? изменились приоритеты?
- Обновить `Dissatisfactions.md` по решениям пользователя

### Шаг 3. Анализ прошлой недели (15–20 мин)

- Собрать коммиты за неделю из всех репозиториев
- Сопоставить с РП прошлого WeekPlan
- Рассчитать `completion_rate`
- Обсудить с пользователем: что удалось, что нет, почему

### Шаг 4. Разбор inbox (10 мин)

- Пройти элементы в `~/ds-strategy/inbox/`
- По каждому: взять в план / capture / удалить / отложить

### Шаг 5. Стратегическая сверка (10–15 мин)

- Сверить фокус месяца из Strategy.md с текущим состоянием
- Проверить: ТОС-месяца снят? гипотеза подтверждается?
- Если первая сессия месяца → месяцная надстройка (см. Session Agenda.md)

### Шаг 6. Формирование WeekPlan (20–30 мин)

- Выбрать РП из результатов месяца + carry-over
- Для каждого: бюджет, день недели, модель (flash/pro)
- Утвердить ТОС недели
- Записать в `~/ds-strategy/current/weekplan-W{N}.md`

### Шаг 7. Утверждение

- Показать пользователю итоговую таблицу
- Запросить корректировки
- После утверждения: закоммитить всё (Dissatisfactions, Strategy, WeekPlan, WP-REGISTRY)

### Шаг 8. DayPlan (если сегодня пн)

- Сформировать DayPlan на сегодня: выбрать РП из WeekPlan на понедельник
- Записать в `~/ds-strategy/current/dayplan-YYYY-MM-DD.md`
- Убедиться, что бюджет на день укладывается в ~5-6 рабочих часов
