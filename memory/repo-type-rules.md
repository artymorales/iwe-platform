# Правила работы по типам репозиториев

## 3 типа репозиториев

| Тип | Что содержит | Source-of-truth? | Примеры |
|-----|-------------|-----------------|---------|
| **Base** | Принципы, форматы, протоколы, скрипты | Да (для формата и правил) | `~/iwe-platform/` |
| **DS (governance)** | Планы, реестры, решения, inbox | Нет | `~/ds-strategy/` |
| **DS (knowledge)** | Captures, drafts, inbox | Нет | `~/ds-knowledge-index/` |

> **Base = платформа.** DS = производные от Base.

## Base (~/iwe-platform/)

**Содержит:** Принципы, форматы, протоколы, скрипты, AGENTS.md.

**Можно:** Уточнять протоколы, обновлять правила, добавлять скрипты под новые нужды.

**Нельзя:** Доменное содержание (→ ds-knowledge-index). Планы и решения (→ ds-strategy).

## DS/governance (~/ds-strategy/)

**Содержит:** Стратегию (Strategy.md), НЭП (Dissatisfactions.md), реестр РП (WP-REGISTRY.md), WeekPlan, DayPlan, inbox, архив.

**Можно:** Добавлять/изменять планы, реестры, решения. Добавлять НЭП.

**Нельзя:** Менять протоколы и правила (→ Base). Хранить скрипты и код.

## DS/knowledge (~/ds-knowledge-index/)

**Содержит:** Захваченные знания (captures/), черновики (drafts/), входящие (inbox/).

**Можно:** Добавлять captures, drafts, инсайты.

**Нельзя:** Хранить планы и задачи (→ ds-strategy). Менять протоколы (→ Base).

## Repository-first режим

Никогда не отвечай из общих знаний без опоры на репозиторий.

1. Найди минимум 1 релевантный файл в репозитории
2. Прочитай его
3. Только потом формируй ответ

Если контекст не найден → скажи «в репозитории нет источника», предложи создать.
